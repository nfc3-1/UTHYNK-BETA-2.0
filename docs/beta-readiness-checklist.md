# UThynk Beta Readiness Checklist

This checklist captures the core flows to verify before beta testing.

## Analytics Funnel

Track the full beta journey:

- User arrived
- Selected category
- Selected question
- Submitted answer
- Received challenge
- Completed reasoning loop
- Returned next day

Goal: beta feedback should be tied to observable behavior, not only opinions.

## First-Session Completion

Verify whether a brand-new user can complete one full reasoning challenge without assistance.

Pass condition:

- User understands what to do.
- User can pick or receive a question.
- User can submit an answer.
- User receives useful UThynk feedback.
- User understands the next step.

## Free-Pass Flow

Verify the conversion path:

- Three free challenges are available.
- Sign-up prompt appears after the free pass is used.
- Progress is preserved after signup.

This is a high-risk conversion point and should be tested carefully.

## Mobile Experience

Test on:

- iPhone
- Android
- Tablet

Verify these remain usable:

- Sidebar / thinking profile
- Thinking tools
- Timeline
- Reasoning lab
- Response input
- Category and question selection

## Empty-State Experience

Test the first-use state:

- XP = 0
- Streak = 0
- No badges
- No history

The app should still feel engaging and purposeful before any data exists.

## Profile Persistence

Verify these persist across sessions:

- XP
- Streak
- Trait
- Completed challenges
- Thinking profile

The user should feel they are building something over time.

## Shareable Moment

Verify the Thinking Snapshot includes:

- Logic score
- Strategy score
- Evidence score
- Trait badge
- Share
- Copy
- Download

Goal: create one clear moment users can show someone else.
